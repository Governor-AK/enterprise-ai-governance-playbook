# Purpose

This document defines responsibility assignments for the governance activities supporting MIP.

It establishes who is:

- **Responsible (R)** for performing an activity;
- **Accountable (A)** for the outcome;
- **Consulted (C)** before significant decisions are made; and
- **Informed (I)** after decisions or activities are completed.

The RACI Matrix operationalizes the Governance Operating Model by assigning governance activities to the appropriate organizational functions.

---

# RACI Definitions

| Code | Meaning |
|------|---------|
| **R** | Responsible – Performs the activity. |
| **A** | Accountable – Ultimately answerable for the outcome. Only one accountable owner should exist for each activity whenever practical. |
| **C** | Consulted – Provides input before completion or approval. |
| **I** | Informed – Kept informed of outcomes or decisions. |

# RACI Principles

The matrix is governed by the following principles:

- Every governance activity has a clearly identifiable accountable owner.
- Responsibility is assigned to the function best positioned to perform the activity.
- Consultation does not transfer accountability.
- Independent assurance remains separate from operational ownership.
- Business ownership remains accountable for AI-enabled business outcomes.
- RACI assignments should be reviewed whenever governance processes materially change.

---

# Governance Boundary

This document assigns activity-level governance responsibility.

It does not:

- define detailed role descriptions;
- establish governance decision authority;
- define governance forums;
- establish escalation pathways;
- define lifecycle procedures;
- create authoritative operational records.

Those responsibilities belong to the related Governance Operating Model artifacts.
